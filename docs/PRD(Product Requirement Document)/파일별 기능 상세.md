# 파일별 기능 상세

이 문서는 DuckPrompt 프로젝트 내 각 주요 파일 및 모듈의 역할과 기능을 설명합니다.

---

## 1. 최상위 파일 및 설정

### 1.1. `main.py`

- **기능**: 프로젝트의 최상위 진입점. `src` 디렉토리를 `sys.path`에 추가하고 `src/app.py`의 `main` 함수를 호출.
- **역할**: Python 인터프리터가 프로젝트 모듈을 찾을 수 있도록 경로 설정 및 애플리케이션 실행 시작.

### 1.2. `src/app.py`

- **기능**: PyQt6 `QApplication`을 초기화하고, `MainWindow` 인스턴스를 생성 및 실행.
- **특이사항**:
  - DPI 스케일링 설정 (Windows 환경).
  - 애플리케이션 아이콘 설정.
  - `MainWindow`가 내부적으로 서비스들을 초기화하고 관리함.
  - 시작 시 DB 로그 정리 시도.

### 1.3. `src/config.yml`

- **기능**: **PostgreSQL DB 초기화 스크립트(`postgres_db_initializer.py`)가 참조하는 초기 설정값 정의 파일.** 애플리케이션 실행 시에는 DB에서 설정을 로드하므로 이 파일은 직접 사용되지 않음.
- **주요 설정**: DB의 `application_config` 테이블 컬럼에 대응하는 초기 기본값 정의 (API 키 포함).

## 2. Core 패키지 (`src/core/`)

핵심 비즈니스 로직과 데이터 모델을 포함합니다.

### 2.1. Pydantic 모델 (`src/core/pydantic_models/`)

- **`app_state.py`**: `AppState` 모델 정의. 애플리케이션의 현재 상태(모드, 프로젝트 폴더, 프롬프트 내용, 체크된 파일, 선택된 LLM/모델명, 첨부 파일 목록 등)를 구조화.
- **`config_settings.py`**: `ConfigSettings` 모델 정의. DB의 `application_config` 테이블 내용을 구조화하고 유효성 검사 수행. API 키 필드는 제거됨 (ConfigService에서 별도 관리). 모델 목록, Gemini 파라미터 필드 포함. 필터링 관련 필드 유효성 검사 로직 포함.

### 2.2. 서비스 (`src/core/services/`)

- **`config_service.py`**: `ConfigService` 클래스. `DbService`를 사용하여 DB에서 설정을 로드, 업데이트하며 `ConfigSettings` 모델 사용. 기본 모델명/Gemini 파라미터 접근 메서드 제공. **메모리에서 사용자 선택 API 키 ID 및 마지막 사용 키 정보 관리.**
- **`db_service.py`**: `DbService` 클래스. PostgreSQL 데이터베이스 연결, 쿼리 실행(설정 로드/저장, API 키 CRUD, 모델 목록/Rate Limit 관리, Gemini 로그 기록/업데이트/정리, API 키 사용량 업데이트/조회, Rate Limit 체크) 담당.
- **`filesystem_service.py`**: `FilesystemService` 클래스. 파일 시스템 관련 작업 처리 (gitignore 로딩, 파일/디렉토리 무시 여부 판단, 디렉토리 트리 생성). `ConfigService` 주입받아 설정 사용. `get_directory_tree`는 파일 크기 정보 및 아이콘 포함.
- **`gemini_service.py`**: LangGraph 워크플로우 빌드 함수(`build_gemini_graph`) 및 Gemini API 호출 노드(`call_gemini`), 응답 처리 노드(`process_response`) 구현.
  - `call_gemini`: 멀티모달 입력 처리, `ConfigService` 및 `DbService` 연동하여 **API 키 선택/회전(사용자 선택 우선, 이후 사용량 기반), Rate Limit 체크, DB 로깅 수행.** `google-generativeai` SDK 직접 사용, 스트리밍 응답 처리 및 오류 처리 강화.
  - `process_response`: Gemini 응답을 XML과 Summary 부분으로 파싱하고 DB 로그 업데이트.
- **`prompt_service.py`**: `PromptService` 클래스. 다양한 종류의 프롬프트(코드 강화, 메타) 생성 로직 담당. 코드 강화 프롬프트 생성 시 첨부 파일 정보 마커 포함.
- **`state_service.py`**: `StateService` 클래스. `AppState` 모델을 사용하여 애플리케이션 상태를 JSON 파일로 저장, 로드, 가져오기, 내보내기 기능 제공. 첨부 파일 정보 직렬화/역직렬화 처리 (데이터 제외). `utils.helpers.get_resource_path` 사용.
- **`template_service.py`**: `TemplateService` 클래스. 프롬프트 템플릿(`.md` 파일)을 로드, 저장, 삭제하는 기능 제공. `utils.helpers.get_resource_path` 사용.
- **`token_service.py`**: `TokenCalculationService` 클래스. GPT(`tiktoken`), Gemini(API), Claude(API) 모델별 토큰 계산 로직 담당.
  - Gemini 토큰 계산 시 멀티모달 입력(텍스트, 이미지, 파일 데이터) 처리. `google-generativeai` SDK의 `count_tokens` 사용.
  - `ConfigService`를 통해 API 키 및 모델명 사용 (DB 기반). API 호출 시 예외 처리 포함.
- **`xml_service.py`**: `XmlService` 클래스. XML 문자열을 파싱하여 `<code_changes>` 태그 내의 파일 생성/수정/삭제 작업을 수행. 파싱 전 XML 문자열 주변의 Markdown 코드 블록(```xml ... ```) 제거 로직 추가. 경로 조작 방지 로직 포함.

### 2.3. LangGraph 상태 (`src/core/langgraph_state.py`)

- **`langgraph_state.py`**: `GeminiGraphState` TypedDict 정의. LangGraph 워크플로우에서 사용되는 상태 변수들을 타입 힌트로 명시. `input_attachments` (첨부 파일 데이터 목록), `selected_model_name`, `log_id` 필드 추가됨.

## 3. UI 패키지 (`src/ui/`)

사용자 인터페이스와 관련된 코드를 포함합니다.

### 3.1. 컨트롤러 (`src/ui/controllers/`)

- **`file_tree_controller.py`**: 파일 트리 뷰 관련 로직 담당 (폴더 선택, gitignore 로드/필터 업데이트, 트리 새로고침, 파일/폴더 이름 변경, 삭제, 체크 상태 변경 처리). `FilesystemService`, `ConfigService` 사용. SettingsDialog와 연동하여 gitignore 관리. 체크 상태 변경 시 상태 표시줄 업데이트 트리거.
- **`main_controller.py`**: 애플리케이션 전반의 흐름 제어 (프로그램 리셋, 상태 표시줄 업데이트 - 문자/토큰 수, API 시간). LLM 선택 및 모델명 변경 처리, `TokenCalculationService` 호출 (첨부 파일 포함, 백그라운드 스레드 사용). 텍스트 변경 시 토큰 레이블 리셋. 첨부 파일 관리(파일 추가, 클립보드 붙여넣기 - 이미지/파일 경로, 선택 제거) 로직 구현. Gemini API 호출 시작 로직.
- **`prompt_controller.py`**: 프롬프트 생성 및 클립보드 복사 관련 로직 담당. `PromptService` 사용. 코드 강화 모드 및 메타 프롬프트 모드 프롬프트 생성 로직 포함. 프롬프트 생성 시 첨부 파일 정보 포함 및 토큰 계산 트리거.
- **`resource_controller.py`**: 리소스 관리 뷰(템플릿/상태 목록) 관련 로직 담당 (목록 로드, 선택 항목 로드/저장/삭제/업데이트). `TemplateService`, `StateService` 사용. 버튼 레이블 동적 변경. 상태 로드 시 `partial_load` 옵션 처리 (마지막 작업 불러오기).
- **`system_prompt_controller.py`**: (함수 형태) 기본 시스템 프롬프트 로드(`apply_default_system_prompt`) 및 선택(`select_default_system_prompt`) 관련 로직 담당. `ConfigService` 연동. 상대/절대 경로 처리.
- **`xml_controller.py`**: XML 파서 실행 로직 담당. `XmlService` 사용. 실행 후 파일 트리 새로고침 트리거.

### 3.2. 모델 (`src/ui/models/`)

- **`file_system_models.py`**: 파일 트리 뷰를 위한 PyQt 모델 정의.
  - `FilteredFileSystemModel`: `QFileSystemModel` 확장, 디렉토리 내용을 재귀적으로 미리 로드.
  - `CheckableProxyModel`: `QSortFilterProxyModel` 확장, 체크박스 기능 추가, gitignore 패턴 기반 필터링, 체크 상태 관리 (재귀적/다중 선택 처리), 파일 크기 표시. `FilesystemService` 사용.

### 3.3. 위젯 (`src/ui/widgets/`)

- **`check_box_delegate.py`**: `QStyledItemDelegate` 확장. 파일 트리 뷰에서 체크박스 영역 클릭 시에만 체크 상태가 토글되도록 처리.
- **`custom_tab_bar.py`**: `QTabBar` 확장. 탭 추가("+"), 중간 클릭으로 탭 닫기, 더블 클릭으로 탭 이름 변경 기능 구현. `tab_manager` 사용.
- **`custom_text_edit.py`**: `QTextEdit` 확장. 서식 없는 텍스트 붙여넣기 기능 등 커스터마이징.
- **`file_tree_view.py`**: `QTreeView` 확장. 드래그 선택 시 체크 상태 토글 기능 추가.
- **`tab_manager.py`**: 보호된 탭 이름 목록 관리 및 탭 삭제/이름 변경 가능 여부 확인 함수(`is_tab_deletable`) 제공.

### 3.4. `main_window.py`

- **기능**: 메인 윈도우 클래스. UI 구성 요소 생성 및 배치 로직은 `main_window_setup_ui.py`로, 시그널 연결 로직은 `main_window_setup_signals.py`로 분리됨.
- **역할**:
  - 서비스 및 컨트롤러 인스턴스 생성 및 주입.
  - 애플리케이션 모드(Code Enhancer/Meta Prompt)에 따른 UI 동적 구성.
  - `AppState` 기반으로 UI 상태 가져오기/설정하기 메서드 제공 (첨부 파일 처리, `partial_load` 옵션 포함).
  - 환경 설정 다이얼로그(`SettingsDialog`) 열기 및 설정 적용 로직 포함.
  - LangGraph Worker 스레드(`GeminiWorker`)를 사용하여 Gemini API 호출 시작 및 응답/오류 처리 슬롯 구현.
  - 첨부 파일 목록 UI(`QListWidget`) 관리 및 업데이트 메서드(`_update_attachment_list_ui`) 제공 (썸네일 표시 포함).
  - 상태 표시줄 API 시간 표시 및 업데이트.
  - **DB 기반 설정 로드 및 적용.**
  - **자동 저장 기능 (QTimer 사용).**
  - **Ctrl+Enter 이벤트 필터링 (사용자 탭에서 '한번에 실행' 후 'Gemini 전송').**

### 3.5. `main_window_setup_ui.py`

- **기능**: `MainWindow`의 UI 위젯(메뉴, 버튼, 탭, 트리 뷰, 상태 표시줄 등) 생성 및 레이아웃 배치 담당. `CustomTextEdit`, `CustomTabBar`, `CheckBoxDelegate`, `FileTreeView` 사용. 모드별 탭 구성 및 위젯 추가. 첨부 파일 관리 그룹(`QGroupBox`) 및 리스트 위젯(`QListWidget`) 추가. 상태 표시줄 API 시간 라벨 추가. LLM 컨트롤(콤보박스, Gemini 파라미터) 상단 이동.
- **변경사항**:
  - **"⏪ 마지막 작업 불러오기" 버튼 추가.**
  - **"💾 현재 작업 저장" 버튼 추가.**
  - **"파일" 메뉴 제거, "환경 설정" 메뉴를 첫 번째 메뉴로 추가.**
  - **UI 레이아웃 조정 (리소스 관리 위치 변경, 스플리터 사용).**

### 3.6. `main_window_setup_signals.py`

- **기능**: `MainWindow`의 UI 위젯 시그널과 해당 컨트롤러 슬롯(또는 `MainWindow` 메서드) 연결 담당. Gemini 전송, 메타 프롬프트 관련 버튼, 설정/상태 메뉴 액션, 텍스트 변경 시그널 연결 포함. 첨부 파일 관리 버튼 시그널 연결. **상태 변경 시그널(`state_changed_signal`) 연결.**
- **변경사항**:
  - **"⏪ 마지막 작업 불러오기", "💾 현재 작업 저장" 버튼 시그널 연결 추가.**
  - **Gemini 파라미터 변경 시그널 연결 제거 (DB 저장 비활성화).**
  - **Ctrl+Enter 단축키 연결 제거 (eventFilter에서 처리).**

### 3.7. `settings_dialog.py`

- **기능**: 환경 설정(DB) 및 현재 프로젝트의 `.gitignore` 파일을 관리하는 다이얼로그 창 UI 및 로직 구현. `ConfigService`, `DbService`와 상호작용. UI에서 설정값을 읽어 `ConfigSettings` 모델로 변환 후 저장 시도 (DB 저장 로직 포함). **API 키 및 사용 가능 LLM 모델 목록 관리 UI 및 로직 추가 (CRUD, 기본 모델 지정, 사용량 표시, 사용자 키 선택).**

## 4. 유틸리티 패키지 (`src/utils/`)

### 4.1. `helpers.py`

- **기능**: 프로젝트 전반에서 사용되는 보조 함수 제공.
- **주요 함수**:
  - `get_project_root()`: 프로젝트 루트 디렉토리 경로 반환 (pathlib 사용).
  - `get_resource_path()`: 리소스 파일 절대 경로 반환 (pathlib 사용).
  - `calculate_char_count()`: 텍스트 길이 계산.

### 4.2. `postgres_db_initializer.py`

- **기능**: PostgreSQL 데이터베이스 스키마(테이블, 함수, 트리거 등)를 생성/업데이트하고, `src/config.yml` 파일에서 초기 설정값을 읽어 `application_config`, `api_keys`, `model_rate_limits` 테이블에 삽입/업데이트하는 스크립트.
- **역할**: 애플리케이션 실행 전 필요한 DB 구조와 기본 데이터를 준비.

### 4.3. `db_migration_script.py`

- **기능**: PostgreSQL 데이터베이스 스키마 마이그레이션 스크립트. `api_key_usage` 테이블의 데이터를 `api_keys` 테이블의 새 사용량 컬럼으로 이전하고, `api_key_usage` 테이블을 삭제.
- **역할**: 이전 버전 DB 스키마에서 현재 스키마로 업데이트.

### 4.4. `notifications.py`

- **기능**: 데스크톱 알림 표시 기능 제공 (Windows 환경에서 `winotify` 사용).
- **역할**: Gemini API 호출 완료/오류 시 사용자에게 알림.

## 5. 기타 파일

- **`app_*.spec`**: PyInstaller 빌드 설정 파일 (AMD64, ARM64). 의존성 및 포함 리소스 업데이트됨 (LangGraph, Pillow, psycopg2-binary, 폰트 등). `hiddenimports`에 `pkg_resources` 포함.
- **`build.bat`**: Windows 환경 빌드 스크립트. 시스템 아키텍처 감지 및 해당 spec 파일 사용.
- **`pyproject.toml`**: 프로젝트 메타데이터 및 빌드/도구 설정 (PEP 621). 의존성 업데이트됨 (`google-generativeai`, `anthropic`, `langgraph`, `Pillow`, `psycopg2-binary`, `winotify` 추가).
